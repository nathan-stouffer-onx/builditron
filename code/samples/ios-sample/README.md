# ios-sample

A SwiftUI sample app that calls into mapitron via Swift C++ interop.

## First-time setup

This project depends on `builditron.xcodeproj`, which is generated by CMake. Because
CMake generates non-deterministic UUIDs, the cross-project dependency cannot be
pre-committed — it must be wired up once by hand after the first configure.

### 1. Configure and generate the CMake project

From the repo root:

```
cmake --preset dest-ios-simulator-xcode
```

This generates `out/build/dest-ios-simulator-xcode/builditron.xcodeproj`.

### 2. Open the sample project

Open `code/samples/ios-sample/ios-sample.xcodeproj` in Xcode.

### 3. Add the CMake-generated project as a sub-project

Drag `out/build/dest-ios-simulator-xcode/builditron.xcodeproj` into the
ios-sample project navigator (top-level, not inside any group).

### 4. Add a target dependency

Select the `ios-sample` target → Build Phases → Target Dependencies → `+` →
choose `mapitron` from the builditron project.

### 5. Link the merged library

In the same target → Frameworks, Libraries, and Embedded Content → `+` →
choose `libmapitron-ios.a` from the builditron project's products.

### 6. Build and run

Select a simulator destination and click play. Xcode will build the mapitron
C++ library first, then compile and link the Swift app.

## After a clean build

If `out/` is deleted and CMake is re-run, the generated project gets new UUIDs.
Xcode will show a missing reference for `builditron.xcodeproj`. Repeat steps
1 and 3–5 to restore the dependency.
